# UI/UX i Design - Testowniko

## 1. Responsywność (Web Responsive Design)
Aplikacja musi wyglądać dobrze zarówno na dużych monitorach, jak i na urządzeniach mobilnych.

### 1.1 Desktop
*   **Szerokość treści:** Maksymalna szerokość kontenera z treścią (np. 800px) wyśrodkowana na ekranie.
*   **Przyciski:** Nie powinny rozciągać się na całą szerokość ekranu. Powinny mieć stałą maksymalną szerokość lub być dopasowane do zawartości.
*   **Układ:** Wykorzystanie wolnej przestrzeni po bokach (marginesy).

### 1.2 Mobile
*   **Szerokość treści:** Wykorzystanie pełnej szerokości ekranu z odpowiednim paddingiem (np. 16px).
*   **Przyciski:** Mogą zajmować większą część szerokości ekranu dla łatwiejszego klikania kciukiem.
*   **Czytelność:** Wszystkie elementy (licznik, pytanie, odpowiedzi) muszą być widoczne bez konieczności nadmiernego przewijania.

## 2. Motywy (Themes)
Aplikacja wspiera dwa tryby:
*   **Jasny (Light Mode):** Białe/jasnoszare tło, ciemny tekst.
*   **Ciemny (Dark Mode):** Ciemne tło, jasny tekst.
Użytkownik powinien móc przełączać motyw w ustawieniach na Ekranie Powitalnym.

## 3. Feedback Wizualny w Quizie
Kolory po wybraniu odpowiedzi:
*   **Zielony (#4CAF50):** Poprawna odpowiedź (wybrana lub nie).
*   **Czerwony (#F44336):** Błędna odpowiedź (wybrana przez użytkownika).
*   **Wyblakły Zielony (#A5D6A7):** Poprawna odpowiedź, której użytkownik nie zaznaczył (tylko w pytaniach wielokrotnego wyboru lub po zatwierdzeniu).

## 4. Ekran po Quizie
*   **Element centralny:** Grafika przedstawiająca kufel piwa (jako nagroda).
*   **Czas:** Wyświetlany w formacie MM:SS.
*   **Akcja:** Duży, wyraźny przycisk "Powrót do menu".
